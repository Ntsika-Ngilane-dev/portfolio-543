# Git Commit Blueprint

This document outlines a clean, professional commit history for the portfolio project. Each entry represents a meaningful milestone so the project can be tracked clearly in Git.

Commit Sequence

1. init: scaffold the portfolio website structure
   - Create the initial HTML, CSS, and JavaScript files
   - Set up the base project folder and assets

2. feat: build the hero section and navigation
   - Add the landing hero experience and sticky navigation
   - Create the modern visual identity for the homepage

3. feat: add the About Me section
   - Include profile image, personal bio, contact details, and role summary
   - Add technology badges for the current stack

4. feat: add downloadable CV action
   - Link the CV PDF from the About section with a clear download button

5. feat: create the project showcase grid
   - Add portfolio cards with screenshots, project titles, and live links

6. feat: wire the contact form to email delivery
   - Connect the form submission to the personal inbox
   - Add subject and hidden form metadata

7. feat: add floating AI-style chat assistant
   - Implement a minimizable pop-up assistant for profile-based queries
   - Add quick replies and WhatsApp contact support

8. style: refine responsive layout and visual polish
   - Improve spacing, typography, button states, and mobile behavior
   - Ensure the interface feels cohesive across screen sizes

9. docs: add professional README and project documentation
   - Document the project purpose, setup, and customization options
   - Include contact and usage guidance

10. chore: organize screenshots and project assets
    - Structure local images and ensure references are consistent
